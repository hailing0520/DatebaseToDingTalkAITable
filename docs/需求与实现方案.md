# 数据库同步钉钉 AI 表格 - 需求与实现方案

## 一、方案可行性结论

**结论：方案可行。** 钉钉开放平台提供了获取 Access Token、获取数据表列表、获取字段列表及**批量插入记录**的完整接口链，配合 Python 读取 SqlServer 与本地 Web 配置页面，可实现“数据库 → 钉钉 AI 表格”的批量同步。实现前需在文档中明确若干缺失项（见下文“需补充与确认”部分）。

---

## 二、需求概述

### 2.1 目标

开发一个 **Python 小程序**，实现：

1. **数据源**：从 SqlServer 数据库读取数据（用户通过**自定义 SQL/视图**输入查询 SQL，先查询出结果集，再在后续步骤中写入 AI 表格）  
2. **数据目标**：通过钉钉开放平台接口，将查询结果**全量批量写入**钉钉 AI 表格  
3. **操作方式**：提供**前台配置页面**（HTML 或独立小窗口），用于配置数据库、钉钉应用、AI 表格及字段映射，并执行同步；**暂不实现登录与权限控制**，先完成基本功能

### 2.2 功能清单

| 序号 | 功能模块 | 描述 |
|------|----------|------|
| F1 | 数据库连接配置 | 选择数据库类型（优先 SqlServer），填写连接信息，支持**测试连接** |
| F2 | 钉钉鉴权 | 根据 AppKey + AppSecret 获取 Access Token（调用钉钉官方接口） |
| F3 | AI 表格与数据表选择 | 用户输入 AI 表格 ID → 获取所有数据表 ID 列表 → 用户选择目标数据表 |
| F4 | 字段拉取与映射 | 根据所选数据表获取全部字段 → 用户选择“SQL 查询结果中的哪些字段”同步到“AI 表格哪些字段” |
| F5 | 数据同步执行 | 将 SQL 查询结果**全量**按映射调用钉钉接口**批量插入**到 AI 表格 |

---

## 三、钉钉接口链（与需求对应）

| 步骤 | 需求对应 | 接口说明 | 官方文档链接 |
|------|----------|----------|--------------|
| 1 | F2 | 获取企业内部应用 Access Token | https://open.dingtalk.com/document/development/obtain-the-access-token-of-an-internal-app |
| 2 | F3 | 根据 AI 表格 ID 获取所有数据表（Sheet） | https://open.dingtalk.com/document/development/api-notable-getallsheets |
| 3 | F4 | 根据数据表 ID 获取该表所有字段 | https://open.dingtalk.com/document/development/api-noatable-getallfields |
| 4 | F5 | **批量插入记录** | https://open.dingtalk.com/document/development/api-notable-insertrecords |
| — | F5 | **记录值格式**（插入时字段值的写法） | https://open.dingtalk.com/document/development/record-value-format |

说明：**插入记录**时，每条记录的字段值必须按照官方《记录值格式》文档进行构造（如文本、数字、日期、人员等类型对应的 JSON 结构）。实现时需同时参考 api-notable-insertrecords 的请求体与 record-value-format 的取值规范。

## 四、实现步骤（建议顺序）

### 阶段一：基础能力

1. **项目与依赖**
   - 使用现有 Python 工程（如 `pyproject.toml`），增加依赖：SqlServer 驱动（如 `pyodbc` 或 `pymssql`）、HTTP 客户端（如 `requests`）、可选 Web 框架（如 `Flask`/`FastAPI`）用于提供前端页面和 API。
   - 确定 Python 版本（当前为 >=3.11）。

2. **钉钉鉴权与接口封装**
   - 实现根据 AppKey、AppSecret 获取 Access Token 的调用（参考官方文档，注意 Token 缓存与过期刷新）。
   - 封装：获取所有数据表、获取数据表字段、**批量插入记录** 三个接口的调用（请求 URL、Header、Body 格式以官方文档为准）。

3. **SqlServer 连接与查询**
   - 实现连接参数配置（主机、端口、库名、用户名、密码等）。
   - 实现“测试连接”逻辑（执行简单查询或 ping）。
   - 实现**自定义 SQL 查询**：用户输入一条查询 SQL（只读，如 `SELECT ... FROM 表/视图`）→ 执行后返回结果集的字段列表 + 数据行（支持分页/限制条数），该结果集将在后续步骤中全量插入 AI 表格。

### 阶段二：字段映射与同步逻辑

4. **数据表与字段选择**
   - 前端：输入 AI 表格 ID → 调用后端 → 展示数据表列表；用户选择数据表 → 调用后端 → 展示该表字段列表。
   - 后端：提供“获取数据表列表”“获取数据表字段”两个 API，内部调用钉钉封装好的接口。

5. **字段映射配置**
   - 前端：展示“数据库字段”与“AI 表格字段”的列表，支持用户勾选并建立映射（数据库字段 A → AI 表格字段 B）。
   - 后端：保存/接收映射配置（如 JSON），供同步时使用。

6. **同步执行与批量写入**
   - 使用用户配置的 SQL 再次执行查询，按映射关系读取结果集（可分批，如每批 100 条）。
   - 将每行数据按官方《记录值格式》文档（record-value-format）转换为钉钉《插入记录》接口要求的字段值结构；若 AI 表格**必填字段**在当行无对应数据，则**报错并中止或跳过该条并记录**（见下文 5.3）。
   - 调用插入接口，支持多批循环；处理限流、部分失败重试与结果汇总（成功/失败条数、错误信息）。

### 阶段三：前台与联调

7. **前台操作页面**
   - 采用 HTML 页面（或独立小窗口）：可选用 Flask/FastAPI 渲染模板 + 静态资源，或前后端分离（前端 HTML+JS 调后端 REST）。
   - 页面包含：数据库连接表单 + 测试连接、**自定义 SQL 输入**、钉钉 AppKey/AppSecret、AI 表格 ID、数据表选择、字段映射、执行同步按钮、执行日志/结果展示。当前**不实现登录与权限**，仅做基本功能。

8. **联调与异常处理**
   - 联调：数据库 → 后端、后端 → 钉钉接口；校验 Token 过期、网络错误、钉钉返回的业务错误码。
   - 异常处理：连接失败、鉴权失败、单条/批量写入失败时的提示与日志；必要时支持“断点续传”或“跳过已存在”策略（若 API 支持）。

## 五、已确认与需补充的内容

以下为已确认的规格，以及实现时仍需查阅或注意的项。

### 5.1 数据源与同步策略（已确认）

| 项 | 说明 | 结论 |
|----|------|------|
| 数据范围 | 同步“整表”还是“自定义 SQL/视图”？ | **采用自定义 SQL/视图**：用户输入一条查询 SQL，先执行得到结果集，再在后续步骤中将该结果集插入到 AI 表格。 |
| 同步策略 | 每次全量覆盖还是按条件增量追加？ | **全量插入**：将 SQL 查询出来的数据全量插入到目标 AI 表格（不做去重、覆盖或增量判断）。 |
| 大数据量 | 单表几十万行时的策略？ | 分批读取 + 分批调用插入接口；单批条数需根据钉钉 insertrecords 限制来定。 |

### 5.2 钉钉侧（已确认 / 待实现时查阅）

| 项 | 说明 | 结论 |
|----|------|------|
| 插入记录接口 | 批量写入的接口与参数。 | 接口：**api-notable-insertrecords**；实现时查阅请求体、单次最大条数。 |
| 记录值格式 | 插入时每条记录的字段值如何书写。 | 以官方《记录值格式》为准：https://open.dingtalk.com/document/development/record-value-format |
| 获取所有字段 | 官方文档链接。 | 已确认：https://open.dingtalk.com/document/development/api-noatable-getallfields |
| 权限与应用类型 | 使用“企业内部应用”还是“第三方/个人应用”？ | 当前按“企业内部应用”+ AppKey/AppSecret 获取 token 设计。 |
| AI 表格 ID 获取方式 | 用户从何处复制“AI 表格 ID”？ | 在说明或界面提示中注明：从钉钉 AI 表格的 URL 或分享/设置中获取。 |

### 5.3 字段映射与类型（已确认）

| 项 | 说明 | 结论 |
|----|------|------|
| 类型兼容 | SqlServer 与 AI 表格字段类型可能不一致。 | 实现时做简单类型转换（如日期统一成字符串格式、数字转文本等），并参考《记录值格式》文档；复杂类型（如附件、人员）需单独约定。 |
| 必填字段无值 | AI 表格必填字段在数据库/SQL 结果中无对应值时如何处理？ | **报错处理**：同步时若发现某条记录在必填字段上无值，则进行报错（可中止整批或跳过该条并记录错误，具体策略在实现时统一为“报错”）。 |

### 5.4 安全与部署（已确认）

| 项 | 说明 | 结论 |
|----|------|------|
| 敏感信息 | 数据库密码、AppSecret 如何存储？ | 仅内存或本地配置文件、不提交到代码库；若提供 Web 界面，建议本地使用或内网部署，避免暴露。 |
| 登录与权限 | 是否需要登录、权限控制？ | **当前不需要**：先实现基本功能，不实现登录与权限控制。 |

## 七、小结

- **可行性**：方案可行，钉钉具备 Token、数据表、字段、插入记录及记录值格式的完整文档与接口链；Python + SqlServer + 简单 Web 前台的组合可满足需求。  
- **实现顺序**：先做钉钉接口封装与 SqlServer 连接（含自定义 SQL 查询），再做字段映射与按《记录值格式》的批量插入逻辑，最后做前台页面与联调。  
- **已明确**：数据源为自定义 SQL 查询结果；同步策略为全量插入；必填字段无值时报错；当前不实现登录与权限；获取所有字段与记录值格式的官方文档已确认并写入上文。  

实现时仅需按官方文档补齐 api-notable-insertrecords 的请求格式与单次条数限制，即可按“实现步骤”分阶段开发与测试。
